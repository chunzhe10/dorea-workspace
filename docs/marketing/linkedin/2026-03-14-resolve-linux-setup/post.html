A few days ago I moved my desktop to Linux. Lower RAM baseline, native Docker, GPU assignment with DRI_PRIME. The engineering benefits were clear. But the real test was creative work.

So I installed DaVinci Resolve on Ubuntu 24.04 for the first time.

It didn't go smoothly. Dependency conflicts, library mismatches, things that should just work on a fresh Ubuntu install but don't. Most of the issues traced back to the same root causes: Wayland and Ubuntu 24.04 specifics that Resolve doesn't expect.

I would have given up. But I had Claude.

Every wall I hit, Claude read the error, searched for known issues, and wrote the fix. Each fix revealed the next issue. Wayland kept being the common thread.

Then came the one that almost beat us both.

Resolve threw "GPU memory is full" with black playback. Claude initially diagnosed it as an out-of-memory issue and we started building a proxy pipeline to work around it. But something didn't add up. I dug deeper and steered us back to the debug logs.

Turns out it wasn't a memory problem at all. It was a Wayland compatibility issue. Together we landed on three environment variables to force Resolve onto X11 instead. Full 4K playback. No proxies needed.

That's the part people miss about working with AI. It's not "ask and receive." It's a collaboration. Claude brings speed and breadth. You bring judgment and direction. Neither of us would have solved that one alone.

I recently found a DaVinci Resolve MCP, and I'm starting to imagine what's possible when an AI can actually interact with your editing timeline. Still brainstorming, but the pieces are coming together.

The stack I built for engineering is now running a full creative workflow. No new hardware. Just the right tools.
