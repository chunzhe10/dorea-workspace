Today I went down a rabbit hole trying to figure out why my laptop keeps crashing.

I've been running VS Code Devcontainers for Corvia development, which means Docker, which on Windows means WSL2. With 40 GB of RAM I assumed I had plenty. I was wrong. OOM crashes, daily.

Between the Devcontainer, a browser, and local LLM experimentation, everything was fighting for memory. My first thought was to order more RAM. But RAM isn't cheap right now, and I wanted to understand the problem before throwing hardware at it.

So I started digging.

Turns out not all operating systems use memory the same way. Ubuntu idles at around 1 GB. Windows 11 can sit at 2-5 GB+ before you open anything. And WSL2 adds its own VM layer on top of that, one that was leaking memory on my machine through vmmem.

On native Linux, Docker just talks to the kernel directly. No VM, no translation layer. That alone changed things.

The other thing I finally explored is the iGPU. Something that's always bugged me. My laptop has two GPUs and the Intel iGPU has always felt like a waste. Even when I'm video editing, everything runs on the RTX 3060 while the iGPU sits there doing nothing.

Linux lets you assign apps to specific GPUs with `DRI_PRIME`. So I moved VS Code and the desktop to the iGPU, and now the RTX 3060 is free for actual work like local LLM inference for Corvia.

One thing that tripped me up is that which GPU your external monitor uses depends on which port it's plugged into. My HDMI is wired to the NVIDIA card, USB-C to Intel. I was getting stuttering from Reverse PRIME, frames copying across GPUs. Having worked with GPU frame copy mechanisms before I knew what was happening, but the fix was simpler than expected. Just swap cables. RTX 3060 dropped to 0%.

Fair note, Windows has per-app GPU settings too through Display settings and NVIDIA Control Panel. I haven't tried it myself so I can't compare directly. Linux's approach works at the driver level which felt more predictable to me, but I'd be curious to hear from anyone who's used the Windows side.

No new hardware. Just understanding what was already there.

Has anyone else gone through something similar? Curious how others handle this.
