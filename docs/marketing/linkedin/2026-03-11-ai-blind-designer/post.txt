I've been using Claude Code to generate UI for my project. LinkedIn carousel PDFs, a VS Code dashboard, brand guidelines — all built from HTML and CSS.

It writes clean code. It understands flexbox, grid, CSS variables. It can read the source file back to me and explain every property.

But it can't see what it built.

And that's where things break. Text overflows containers. Diagrams overlap. Elements that should be side by side end up stacked on top of each other. A stat box covers the slide title. A bar chart label flies off the edge. The code looks right. The output doesn't.

I'd fix one thing, ask for another change, and something else would break. It's like working with a designer who's writing CSS blindfolded — technically skilled, but missing the one thing that matters: visual feedback.

A human designer writes code, looks at the result, adjusts. That feedback loop is so natural we forget it exists. AI doesn't have it. It writes the CSS and moves on, never seeing whether the div actually landed where it was supposed to. This is the gap. Not intelligence, not coding ability — perception.

So I gave it eyes.

Playwright MCP lets Claude Code launch a browser, render the page, and take a screenshot of what it built. Now it can see the overflow. It can spot the overlap. It can notice that the grid columns collapsed. Same AI, same code generation ability. But now with a feedback loop. The difference was immediate.

This is what I keep learning about working with AI: the model is rarely the bottleneck. The context is. Give it the right tools to close the loop, and it goes from frustrating to genuinely useful.

If you're using AI to generate any kind of visual output — slides, dashboards, emails, PDFs — give it a way to see what it's producing. It's not perfect, but the improvement is significant.
