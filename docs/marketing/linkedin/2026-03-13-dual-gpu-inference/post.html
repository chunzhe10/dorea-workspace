Last post I moved my desktop and VS Code to the Intel iGPU so the RTX 3060 could focus on inference. That helped, but the iGPU was still just pushing pixels. Wasted compute.

So I put it to real work. Corvia's inference server runs two GPU-hungry workloads: embeddings and LLM chat. Both were fighting over the RTX 3060. Now embeddings run on the Intel iGPU via ONNX Runtime + OpenVINO, and LLM stays on NVIDIA via llama.cpp + CUDA. One Rust binary, two GPUs, neither blocking the other.

The key insight is that ONNX Runtime sessions are independent. Each session gets its own hardware backend, and they run in parallel. CUDA, OpenVINO, CPU, all concurrently. You just pick the backend at runtime.

Getting OpenVINO onto Debian Trixie was messier than expected. Wrong drivers installed, missing dependency chain, library in the wrong path. The kind of stuff that used to take an afternoon, but Claude handled it in minutes. (Slide 5 has the full breakdown.)

The part I'm proud of is the devcontainer. Open the project and both GPUs are ready. A host-side script detects hardware before the container builds, generates the right Docker Compose override, and the inference server auto-assigns backends at startup. Falls back to CPU if something's missing, but tells you. No silent degradation.

And it all dogfoods itself. Corvia is organizational memory for AI agents, and the AI assistant building it uses corvia's own knowledge base from the first keystroke.
